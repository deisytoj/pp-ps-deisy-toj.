Frameworks

Qué son, ventajas/desventajas y ejemplos populares.

Concepto

Un framework es una estructura de trabajo reutilizable que facilita el desarrollo al ofrecer bibliotecas y convenciones para resolver problemas comunes (por ejemplo, control de rutas, acceso a datos, plantillas).

Ventajas

  • Productividad: acelera el desarrollo con componentes ya hechos.
  • Mantenimiento: promueve buenas prácticas y arquitectura consistente.
  • Seguridad: muchos frameworks ofrecen protecciones integradas.

Desventajas

  • Curva de aprendizaje: hay que aprender la filosofía del framework.
  • Restricciones: pueden imponer estructuras que no se ajusten a todos los casos.
  • Tamaño/overhead: algunos añaden peso o complejidad innecesaria.

Frameworks Frontend más usados

  • React (biblioteca con ecosistema), Angular, Vue.js, Svelte.

Frameworks Backend más utilizados

  • Express (Node.js), Django (Python), Flask (Python), Spring Boot (Java), Laravel (PHP), ASP.NET (C#).
